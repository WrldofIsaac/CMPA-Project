So pretty much for my website track wave, I decided to keep things super simple for this project. Instead of trying to make some crazy full-blown app with all the bells and whistles, I just wanted to get the basic stuff working. I didn’t even bother with a server or anything complicated, because I tried it and worked for a little bit, but it was just too much work to configure, so I just focused on making sure the buttons, tabs, and simple interactions did what they were supposed to do. And, yeah, I kinda threw all my code into one script file at first because it seemed easier to manage while I was still figuring things out. Later on I decided it was smart to divide them and conquer.

The cool part was actually getting the main features to work. Once I figured out how to link the buttons to actions and make stuff happen when you click, it felt like a real deal. I was pretty excited about the parts where the app responded to what you did, like switching tabs, popping up content or even the api system. It wasn’t anything fancy, but seeing it all come together and work how I planned was pretty satisfying.

If I could do it again, I’d totally be more organized. One of my biggest headaches was dumping everything into that one script file—it made it a pain to keep track of what was going on. I’d split up my code into different sections and files, like keeping related functions together instead of a total free-for-all. At the same time I also had problems where I would double everything which would cause code to break but I got over it and started to pay attention more. Plus, I’d probably plan a bit more before diving in, so I don’t have to backtrack and fix structural stuff later. Being organized from the start would save a ton of time and make debugging way easier.

I learned a lot from this, especially how tiny details can make or break things. I got way better at understanding how JavaScript talks and interacts to HTML and how user actions trigger code. And how debugging Huge part of it. Reading those error messages carefully actually pays off because if I didn't I would not know where to start. I also realized I jump into building without a plan, which gets messy and haywire, but it’s something I can fix with practice and repetition. Overall, this project helped me nail the basics and showed me what I need to focus on next time.
